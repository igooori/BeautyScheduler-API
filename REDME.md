# 📅 BeautyScheduler API

**BeautyScheduler** — это профессиональный микросервис на FastAPI для управления записями к частному мастеру. Проект создан для отработки навыков проектирования БД, сложной бизнес-логики и автоматизированного тестирования.

---

## 🛠 Технологический стек

* **Язык:** Python 3.10+
* **Framework:** FastAPI (Асинхронный)
* **Database:** PostgreSQL + SQLAlchemy 2.0 (Async)
* **Validation:** Pydantic v2
* **Testing:** Pytest + HTTPX

---

## 📋 Техническое задание (Бизнес-логика)

### 1. Модель данных (Booking)
- **client_name**: Строка (Имя клиента).
- **start_time**: Дата и время начала.
- **end_time**: Дата и время конца (Расчитывается автоматически: `start_time + 1 час`).

### 2. Правила создания записи (`POST /bookings`)
* ✅ **Только будущее:** Нельзя создать запись на время, которое уже прошло.
* ✅ **Никаких накладок:** Если слот занят — API выдает ошибку `400 Bad Request`.
* ✅ **Кратность часу:** Мастер принимает только в начале часа (10:00, 11:00 и т.д.).
* ✅ **Рабочее время:** Запись возможна только в интервале с **10:00 до 18:00**.

### 3. Правила отмены (`DELETE /bookings/{id}`)
* ⚠️ **Запрет на позднюю отмену:** Если до начала записи осталось **меньше 2 часов**, удаление запрещено (Status 403).

### 4. Расписание (`GET /bookings/available`)
* 🔍 Возвращает список только свободных часовых слотов на выбранную дату.

## 🧪 План тестирования (QA)

*Цель: Покрытие логики тестами > 70%.*

| Тест-кейс | Описание | Ожидаемый результат |
| :--- | :--- | :--- |
| `test_create_success` | Обычная запись на свободное время | Status 201 |
| `test_collision` | Попытка записаться на то же время | Status 400 |
| `test_past_date` | Попытка записи на вчера | Status 400 |
| `test_late_cancel` | Отмена за 30 минут до начала | Status 403 |
| `test_available_slots` | Проверка фильтрации занятых окон | Список без занятого часа |
